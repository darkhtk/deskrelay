# DeskRelay Self 사용성 평가

## 총평

현재 self 기준으로 냉정히 보면 일반 사용자용 앱은 아직 아니다. 파워유저용 도구로는 꽤 강하다.

| 기준 | 점수 |
|---|---:|
| 개발자/파워유저 기준 | 6.5 / 10 |
| 일반 사용자 기준 | 3 / 10 |
| 오픈소스 self-host 도구 기준 | 7 / 10 |

가장 큰 문제는 기능이 부족한 것이 아니라, 사용자가 머릿속에 들고 있어야 하는 개념이 너무 많다는 점이다.

`서버 PC`, `등록할 PC`, `connector daemon`, `Site token`, `daemon token`, `Tailscale IP`, `workspace root`, `login task`, `Claude CLI 세션`, `권한`, `스킬`, `CTX`, `Session/Week usage`가 한 번에 등장한다. 기능적으로는 흥미롭지만, UX 관점에서는 진입 장벽이 높다.

## 좋은 점

- 브라우저에서 자기 PC의 Claude CLI를 다루는 컨셉은 강하다.
- 여러 디바이스 전환, 세션 목록, 권한/스킬 확인, 이미지 렌더링은 차별점이 있다.
- self-host 방향은 명확하다. 내 PC에 내 서버, 내 디바이스 등록이라는 모델은 납득 가능하다.
- 최근 등록/중복/실패 경로 테스트가 들어가면서 기술적 신뢰도는 올라갔다.
- 상태 표시, 사용량 표시, 권한/스킬 탭은 고급 사용자에게 실제 가치가 있다.

## 나쁜 점

- 설치 UX가 여전히 어렵다. Git, Bun, PowerShell 실행 정책, Tailscale, 방화벽 중 하나만 걸려도 막힌다.
- 다른 PC 등록은 아직 앱스럽지 않다. 명령어를 복사해서 실행하는 방식은 오픈소스 도구로는 괜찮지만, 일반 앱 UX는 아니다.
- 서버 PC와 등록 대상 PC가 같은 경우/다른 경우의 설명이 아직 사용자의 직관에 완전히 붙지 않는다.
- 오류 메시지는 예전보다 나아졌지만, 사용자가 그래서 내가 뭘 해야 하지?를 매번 즉시 알 정도는 아니다.
- 기능 밀도가 높다. 사이드바에 세션/권한/스킬/사용량/설정/디바이스가 모이면 강력하지만, 처음 보는 사람에게는 복잡하다.
- 실제 신뢰를 좌우하는 login task, 재부팅 복구, 방화벽, Tailscale 미설치, 다른 PC 등록은 아직 수동 검증 의존도가 크다.

## 가장 위험한 사용성 결함

1. 등록 명령이 실패했을 때 사용자가 원인을 분류하기 어렵다.

   예: 서버 접근 실패인지, daemon local bind인지, 방화벽인지, token 문제인지, Tailscale 문제인지.

2. 설치 완료와 실제 사용 가능 상태가 다르다.

   사용자는 설치가 끝났다고 생각하지만, 서버에서 접근 가능해야 진짜 완료다. 이 차이를 UX가 아주 강하게 알려줘야 한다.

3. self-host인데도 제품처럼 보이는 부분과 개발도구처럼 보이는 부분이 섞여 있다.

   UI는 꽤 앱 같지만 설치/복구는 CLI 도구다. 이 간극이 혼란을 만든다.

## 현재 잘 맞는 사용자

- 개발자
- AI 도구를 많이 쓰는 사용자
- Tailscale, Git, CLI를 이해하는 사용자

## 애매한 사용자

- Unity/게임 개발자는 가능하다.
- 단, 네트워크나 PowerShell에 약하면 막힌다.

## 안 맞는 사용자

- 비개발 일반 사용자
- 명령어 실행, 방화벽, Tailscale 설정을 피하고 싶은 사용자

## 가장 먼저 개선해야 할 것

1. 설치 후 사용 가능 여부를 한 화면에서 판정해야 한다.

   서버 실행됨, 토큰 있음, 내 PC 등록됨, 외부 접근 가능, Claude CLI 사용 가능을 체크리스트로 보여줘야 한다.

2. 다른 PC 등록 명령을 더 강하게 만들어야 한다.

   실패 원인별로 메시지를 분리하고, 성공 시 서버 목록에 반영됐는지까지 확인해야 한다.

3. 설정 다이얼로그의 디바이스 탭을 등록/상태/복구 중심으로 재정리해야 한다.

   사용자는 디바이스가 왜 안 되는지와 무엇을 눌러야 하는지만 보면 된다.

4. 안내 문서는 더 짧아져야 한다.

   정확성은 좋아지고 있지만, 초심자 입장에서는 여전히 읽는 부담이 크다. 서버 PC에서 1개, 다른 PC에서 1개 수준으로 내려가야 한다.

## 결론

지금 앱은 아이디어와 기능의 힘은 분명하지만, 사용성은 아직 친절한 앱이 아니라 강력한 오픈소스 도구에 가깝다.

다만 방향은 맞다. 설치, 등록, 상태 진단만 더 단단하게 만들면 개발자용 self-host 도구로는 충분히 설득력 있는 수준까지 갈 수 있다.
