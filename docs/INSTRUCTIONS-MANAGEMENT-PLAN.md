# 지침 관리 개선 계획

DeskRelay의 지침 관리는 두 축으로 나눈다.

- 디바이스 지침: 선택한 PC의 Claude Code 전역/관리 정책 지침이다. 설정 다이얼로그에서 다룬다.
- 작업 폴더 지침: 선택한 대화 세션의 cwd에 묶인 프로젝트 지침이다. 채팅 사이드바의 지침 탭에서 다룬다.

## 동작 표

| 지침 종류 | 파일 위치 | 종속 기준 | 초기화 | 디바이스 선택 시 | 대화 선택 시 | 저장 가능 |
|---|---|---|---|---|---|---|
| 사용자 전역 | `~/.claude/CLAUDE.md` | 디바이스 | 파일 없으면 빈 상태로 표시 | 선택된 디바이스의 로컬 파일을 다시 로드 | 영향 없음 | 가능 |
| 관리 정책 | OS 관리 위치의 `CLAUDE.md` | 디바이스 | 파일 없으면 빈 상태, 읽기 전용 | 선택된 디바이스 기준으로 다시 로드 | 영향 없음 | 불가 |
| 프로젝트 지침 | `cwd/CLAUDE.md` | 선택된 대화의 `cwd` | 대화/cwd 없으면 로드하지 않음 | 새 디바이스의 선택 대화/cwd 기준으로 다시 로드 | 선택한 대화의 `cwd`로 즉시 로드 | 가능 |
| 프로젝트 내부 지침 | `cwd/.claude/CLAUDE.md` | 선택된 대화의 `cwd` | 대화/cwd 없으면 로드하지 않음 | 새 디바이스의 선택 대화/cwd 기준으로 다시 로드 | 선택한 대화의 `cwd`로 즉시 로드 | 가능 |
| 개인 로컬 지침 | `cwd/CLAUDE.local.md` | 선택된 대화의 `cwd` | 대화/cwd 없으면 로드하지 않음 | 새 디바이스의 선택 대화/cwd 기준으로 다시 로드 | 선택한 대화의 `cwd`로 즉시 로드 | 가능 |

| 이벤트 | 전역/관리 지침 | 프로젝트/cwd 지침 |
|---|---|---|
| 앱 초기 진입 | 자동으로 강제 로드하지 않음. 설정 다이얼로그의 지침 탭에서 로드 | 대화가 선택되지 않았으면 로드 대상 없음 |
| 디바이스 선택 | 해당 디바이스의 `~/.claude/CLAUDE.md`, 관리 정책 위치를 다시 조회 | 선택된 대화가 있으면 그 디바이스의 해당 `cwd` 기준으로 다시 조회 |
| 대화 선택 | 변화 없음 | 선택한 세션의 `cwd` 기준으로 `CLAUDE.md`, `.claude/CLAUDE.md`, `CLAUDE.local.md` 로드 |
| 저장 | 선택된 디바이스의 실제 파일에 저장 | 선택된 디바이스 + 선택된 대화 `cwd` 아래 실제 파일에 저장 |
| 삭제/초기화 | 사용자 전역은 삭제 가능, 관리 정책은 불가 | 각 cwd 지침 파일 삭제 가능 |

## 개선 목표

1. 설정 다이얼로그에는 대화와 무관한 디바이스 지침만 남긴다.
2. 채팅 사이드바에는 선택한 대화/cwd에 종속되는 지침만 둔다.
3. 파일이 없으면 오류가 아니라 "파일 없음" 상태로 보여준다.
4. 지침 API가 404를 반환하면 캐시 문제가 아니라 오래된 서버/connector 문제임을 명확히 안내한다.
5. 저장은 expected hash를 함께 보내 디스크의 외부 변경을 덮어쓰지 않게 한다.

## 실행 내용

- 사용자 전역 지침 `~/.claude/CLAUDE.md`는 설정 다이얼로그에서 편집, 저장, 삭제할 수 있게 한다.
- 관리 정책 지침은 설정 다이얼로그에서 읽기 전용으로만 표시한다.
- 프로젝트 지침 3종은 선택한 대화의 cwd가 있을 때만 사이드바 지침 탭에서 편집한다.
- 지침 조회 404 메시지에는 서버와 connector를 최신 코드로 재시작하라는 복구 안내를 붙인다.

## 테스트 관점

| 테스트 | 기대 결과 |
|---|---|
| 설정 지침 탭에서 디바이스 변경 | 사용자 전역/관리 정책 지침이 새 디바이스 기준으로 다시 로드된다 |
| 사용자 전역 지침 저장 | 선택 디바이스의 `~/.claude/CLAUDE.md`에 저장되고 상태가 저장됨으로 바뀐다 |
| 사용자 전역 지침 삭제 | 파일이 삭제되고 상태가 파일 없음으로 바뀐다 |
| 대화 미선택 상태에서 사이드바 지침 탭 | 세션을 선택하라는 안내만 보인다 |
| 대화 선택 후 사이드바 지침 탭 | 선택 세션 cwd의 세 지침 파일이 로드된다 |
| 오래된 서버 또는 daemon | 404 원인과 최신 코드 재시작 안내가 같이 표시된다 |
