# DeskRelay Manual

이 문서는 DeskRelay 화면을 빠르게 이해하기 위한 스크린샷 매뉴얼입니다. 설치 명령과 문제 해결은 [README](../README.md)를 기준으로 보세요.

## 메인 화면

현재 저장소에는 메인 화면 스크린샷이 아직 포함되어 있지 않습니다. 메인 화면에서 확인할 핵심은 다음입니다.

- 현재 브라우저가 열린 기기가 `서버 PC`, `등록된 디바이스`, `미등록 디바이스`, `모바일` 중 어디에 해당하는지 상태 라벨로 확인합니다.
- 서버 PC에서는 `시작하기`로 채팅 화면에 들어갑니다.
- 다른 PC를 추가하려면 메인 화면의 `다른 PC 등록 명령`을 통째로 복사해 제어하려는 PC의 PowerShell에서 실행합니다.
- 등록 명령에는 서버 URL과 Site token이 포함되어 있어야 하며, 실행이 끝나면 대상 PC가 디바이스 목록에 나타납니다.
- 서버 URL, Site token, 제거 명령, 상태 진단 안내는 메인 화면에서 확인하는 것이 기준입니다.
- 서버가 삭제되었거나 현재 PC가 더 이상 등록 대상이 아니면, 메인 화면의 안내에 따라 제거 명령을 실행합니다.

## 새 채팅과 작업 폴더

![새 채팅과 작업 폴더 선택](assets/screenshots/new-chat-workspace.png)

- 왼쪽 상단에서 현재 사용할 디바이스를 선택합니다.
- `+` 버튼으로 새 채팅을 시작합니다.
- 새 채팅은 작업 디렉토리를 먼저 고릅니다.
- 목록에 보이는 폴더는 현재 디바이스의 workspace roots 기준입니다.
- 컴포저 위 상태줄은 현재 입력 가능한지, Claude가 대기 중인지, 권한 확인 전인지 같은 핵심 상태를 보여줍니다.

## 채팅 세션

![채팅 세션](assets/screenshots/chat-session.png)

- 왼쪽 세션 탭에서 기존 Claude 세션을 선택합니다.
- 상단에는 현재 세션, 모델, 권한 모드 같은 현재 대화 정보를 표시합니다.
- 오른쪽 대화 영역은 선택한 세션의 transcript를 보여줍니다.
- 상단 사용량 표시는 Session과 Week 기준 남은 사용량과 reset 시간을 보여줍니다.

## 연속 지시 큐잉

![연속 지시 큐잉](assets/screenshots/queued-prompts.png)

- 컴포저에서 메시지를 연속으로 보내면 순서대로 큐에 들어갑니다.
- 앞 요청이 끝난 뒤 다음 요청이 같은 세션에 이어서 실행됩니다.
- 사용자는 여러 지시를 미리 넣어둘 수 있지만, 실제 Claude 실행은 순차 처리됩니다.

## 권한 탭

![권한 탭](assets/screenshots/permissions-sidebar.png)

- 권한 탭은 현재 선택한 디바이스와 현재 선택한 세션 작업 폴더에 종속됩니다.
- `User settings`, `Project settings`, `Project local settings`를 나누어 보여줍니다.
- 허용된 권한은 개별 삭제할 수 있습니다.
- 아래쪽의 추가 가능한 권한을 누르면 `Bash(*)`, `Read(*)` 같은 권한을 추가할 수 있습니다.
- 다음 실행 권한 모드는 여기서 선택합니다.

## 스킬과 슬래시 명령

![스킬 탭](assets/screenshots/skills-sidebar.png)

- 스킬 탭은 현재 디바이스의 Claude 환경과 현재 세션 작업 폴더 기준으로 로드됩니다.
- Claude 기본 스킬과 사용자가 추가한 스킬은 색으로 구분합니다.
- 슬래시 명령도 같은 탭에서 확인합니다.
- 컴포저에서 `/`를 입력하면 사용 가능한 명령과 스킬 후보가 나타납니다.

## 설정: 일반

![설정 일반](assets/screenshots/settings-general.png)

- 전체 업데이트와 서버 업데이트를 실행합니다.
- 테마, 채팅 글자 크기, 표시 옵션처럼 현재 브라우저에 가까운 설정을 다룹니다.
- Session, Week, CTX 표시 여부를 켜고 끌 수 있습니다.
- 세션 로드 개수와 메시지 전송 후 스크롤 동작도 여기서 조정합니다.

## 설정: 디바이스

![디바이스 설정](assets/screenshots/settings-devices.png)

- 등록된 디바이스를 확인하고 선택합니다.
- 디바이스 라벨과 기본 작업 디렉토리를 저장합니다.
- 다른 PC 등록 명령은 메인 화면 wizard가 담당합니다.
- 디바이스 제거는 서버 목록 삭제뿐 아니라 가능한 경우 해당 PC의 connector 설치 상태도 함께 정리합니다.

## 설정: 연결 진단

![연결 진단](assets/screenshots/settings-diagnostics.png)

- 선택한 디바이스가 실제로 사용 가능한지 점검합니다.
- 서버, connector daemon, Claude 모듈, workspace 상태를 나누어 보여줍니다.
- 단순히 “오프라인”으로 뭉개지 않고 어느 단계가 막혔는지 확인하는 화면입니다.
- 문제가 있으면 여기서 먼저 현재 상태와 마지막 오류를 확인하세요.
