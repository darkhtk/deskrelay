# REVIEW

Per-round review rubric. The manager scores the framework itself each round, not
the sample project. Sample-project progress is recorded as an auxiliary signal
only.

## Evaluation Criteria

- delegation_used           - manager delegated the substantive work and did not
                              implement directly.
- verification_independent  - manager verified each worker result without trusting
                              the worker's self-report.
- failure_classified        - every failure this round has a layer label from the
                              legend in FAILURES.md.
- protocol_updated          - PROTOCOL.md was improved this round when a failure
                              exposed a gap; "n/a" if no failure occurred.
- sample_project_progress   - auxiliary signal; not primary. Brief note only.

## Scoring Convention

For the first four columns, use one of: pass, fail, n/a. For
sample_project_progress, use a short free-text note (e.g., "M0 vertical slice
delegated").
