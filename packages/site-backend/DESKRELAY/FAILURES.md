# FAILURES

Failure log for the orchestration lab. Every failure observed by the manager during
verification or by a worker during execution is appended here with a layer label.

Layer legend (use one of these exact values in the layer column):

- server        - upstream service the CLI talks to.
- connector     - transport between the CLI and the server.
- daemon        - any local background process the CLI relies on.
- Claude CLI    - the Claude command-line tool itself.
- worker CLI    - the wrapper or shell invoking the worker.
- workspace     - filesystem state, missing directories, permissions on the workspace.
- permission    - allow/deny policy on a tool, path, or command.
- network       - DNS, proxy, TLS, connectivity.
- repository    - git state, missing files, branch divergence.
- protocol      - a rule in PROTOCOL.md that was insufficient or ambiguous.
- prompt        - the worker prompt itself was malformed or under-specified.
- verification  - a verification step was missing, skipped, or wrong.
- timeout       - a worker exceeded its timeoutMs or stayed idle past 2x timeoutMs and was cancelled.

Status values: open / resolved / deferred / superseded.
